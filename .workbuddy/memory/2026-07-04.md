## 修复 SAN 规范问题 (RFC 5280)

CertUtils 签发证书时 SAN 字段的 DNS 与 IP 区分问题。

### 问题
之前所有 SAN 条目（包括 IP 地址）都被写为 `GeneralName.dNSName`，违反 RFC 5280，导致 Chrome/Firefox/Edge 等现代浏览器在 IP 直连时校验失败。

### 改动
- **CertUtils.java**：重写 SAN 解析逻辑
  - 支持 `DNS:xxx` → `GeneralName.dNSName`
  - 支持 `IP:xxx` → `GeneralName.iPAddress`（使用 `InetAddress.getByName()` 解析 + `DEROctetString` 封装字节）
  - 支持 `EMAIL:xxx` → `GeneralName.rfc822Name`、`URI:xxx` → `GeneralName.uniformResourceIdentifier`
  - 无前缀时自动识别 IP（IPv4 / IPv6）还是域名，分配正确的 GeneralName 类型
  - 解析异常时安全退化为 dNSName
- **CertPanel.java**：SAN 输入框标签改为 `"SAN (DNS:xxx, IP:xxx, 多个用逗号分隔)："`，示例值改为 `"DNS:example.com, DNS:*.example.com, IP:192.168.1.1"`
- 编译：**BUILD SUCCESS**（25.6s）
