# Java 工具箱全局工作台布局设计

## 背景

Java 工具箱是基于 Java Swing 和 FlatLaf 的桌面客户端，当前约有 43 个工具。主窗口使用“顶部分类页签、分类内左侧工具列表、右侧 CardLayout 内容区”的两层导航。该结构可以工作，但分类和工具被拆成两个导航区域，顶部标题、搜索、主题、语言和分类页签共同占用纵向空间；不同窗口尺寸下，固定宽度和重复层级也降低了工具定位效率。

本轮只调整全局导航、顶栏和共享布局规则。所有工具、主题、语言、搜索和状态监控能力必须保留，具体工具面板的业务逻辑不在本轮重排。

## 目标

1. 将分类与工具统一到一个可预测的导航区域，缩短工具定位路径。
2. 形成紧凑但不拥挤的桌面工作台，给工具内容区释放更多可用空间。
3. 在默认窗口和最小窗口尺寸下保持导航及核心操作可达。
4. 保留所有工具实例和页面状态，不因导航切换或搜索而重建内容。
5. 保持 FlatLaf 全主题、中英文切换、键盘操作及状态栏功能完整。
6. 建立可测试的导航模型和共享布局边界，避免继续把全部逻辑堆入 `MainFrame`。

## 非目标

- 不删除、合并或隐藏任何现有工具。
- 不改变工具面板内部的业务流程、网络行为、数据格式或持久化逻辑。
- 不为全部工具增加图标，也不引入新的图标资源体系。
- 不进行网页化改造，不采用大卡片、悬浮抽屉或装饰性动画。
- 不在本轮重排 Kafka、K8s、数据库等复杂管理页面的内部结构。

## 选定方案

采用统一工作台式侧边导航。主窗口由左侧导航、右侧工具区域和底部状态栏组成。左侧导航同时承载应用标题、全局搜索、分类和工具；右侧顶部为紧凑的当前工具栏，中央继续使用 CardLayout 承载现有工具视图。

```text
┌──────────────侧边栏──────────────┬──────────────当前工具栏──────────────┐
│ Java 工具箱        [收起]         │ [展开导航]  加密安全 / 摘要与编解码   │
│ [搜索全部工具……]                  │                  [主题] [语言]       │
│                                  ├─────────────────────────────────────┤
│ ▼ 加密安全                        │                                     │
│   摘要与编解码                    │                                     │
│   对称加密                        │           当前工具内容区             │
│   非对称加密                      │          保留原 CardLayout           │
│ ▶ 转换编码                        │                                     │
│ ▶ 开发调试                        │                                     │
│ …                                │                                     │
├──────────────────────────────────┴─────────────────────────────────────┤
│ 就绪 | JDK | CPU | 内存                                                  │
└────────────────────────────────────────────────────────────────────────┘
```

## 主窗口结构

### 侧边栏

- 默认宽度为 228px，允许用户拖动调整，限制在 190px 至 320px。
- 支持完全收起；收起状态和宽度通过 `ConfigManager` 保存。
- 顶部显示应用名称、收起按钮和全局搜索框。
- 中部使用可滚动的分组工具树，保持现有工具和分组顺序。
- 分类节点可展开或收起，展开状态持久化。
- 工具名称过长时截断显示，并通过 tooltip 提供完整名称。

### 当前工具栏

- 左侧提供展开导航按钮，并显示“分类 / 当前工具”。
- 右侧保留主题和语言选择功能。
- 工具名称空间不足时优先截断当前位置，主题和语言控件保持可操作。
- 工具栏保持紧凑，不增加大标题、说明卡片或装饰元素。

### 内容区

- 继续复用现有 `ToolPanel#getView()` 和 `CardLayout`。
- 工具视图仍然惰性创建，并且每个工具只创建一次。
- 导航切换、搜索过滤、语言切换和主题切换均不销毁工具视图。
- 本轮不对所有工具增加全局滚动容器，避免复杂页面出现嵌套滚动条。

### 状态栏

- 完整保留当前就绪状态、JDK、CPU 和内存信息。
- 状态栏继续位于窗口底部，不迁移到导航或工具栏。

## 交互设计

### 导航

- 单击分类节点展开或收起。
- 单击工具节点立即显示对应 CardLayout 页面，并更新当前位置。
- 当前工具使用主题提供的选中态，同时保留键盘焦点边框，不能只依赖颜色表达状态。
- 应用记住最后打开的稳定工具 ID，下次启动恢复该工具。
- 如果保存的工具 ID 已失效，则回退到第一个可用工具。

### 搜索

- `Ctrl+K` 聚焦全局搜索。
- 输入内容后直接过滤侧栏中的分类和工具，不再使用独立弹出菜单。
- 搜索匹配继续覆盖稳定名称、本地化标签、分组和现有搜索关键词。
- 搜索结果保持原始分组和工具顺序，并自动展开包含结果的分类。
- 方向键移动结果，`Enter` 打开工具，`Esc` 清空搜索并恢复完整导航。
- 无结果时显示不可选中的“未找到工具”状态，不显示空菜单或错误对话框。
- 过滤过程不改变工具实例，也不覆盖用户原先保存的分类展开状态。

### 侧栏缩放和收起

- 主体使用水平 `JSplitPane` 或等价的可调宽结构承载侧栏和内容区。
- 用户拖动结束后保存经过上下限约束的侧栏宽度。
- 收起侧栏后，当前工具栏的展开按钮始终可见。
- 展开时恢复上次有效宽度，不创建临时悬浮抽屉。

### 主题和语言

- 主题切换继续使用现有 `ThemeManager`，导航背景、边框、选中态和焦点态均从 `UIManager` 读取。
- 不为浅色或深色主题硬编码背景及文字颜色。
- 中英文切换后原地刷新分类、工具、搜索占位文本和当前位置。
- 本地化刷新依赖稳定工具 ID，不通过翻译后的字符串定位工具。
- 语言和主题切换后保留当前工具、搜索外的工具状态及侧栏尺寸。

## 视觉与密度规则

- 使用 4、8、12、16px 的共享间距节奏。
- 常规按钮、文本框和下拉框维持至少 32px 高度。
- 侧栏行高应适合连续浏览，工具行目标为 32px 至 34px，分类行可略高并使用更明确的字重。
- 内容区保留现有 `UIUtils.CONTENT_PADDING` 机制，本轮只统一全局壳层边距，不对全部页面做机械重写。
- 遵循 FlatLaf 标准组件行为，不定制滚动条，不使用渐变文字、玻璃效果或网页式大卡片。
- 默认物理场景为开发者或运维人员在普通办公光线下，将工具箱与 IDE、终端或浏览器并排使用；主题继续尊重用户当前选择。

## 组件和职责边界

### `MainFrame`

- 创建工具实例并组装工作台壳层。
- 连接侧栏选择事件、当前位置和 CardLayout。
- 管理顶层窗口、主题、语言和状态栏。
- 不直接实现搜索过滤和树模型细节。

### `ToolNavigationModel`

- 纯 Java 模型，不依赖可见 Swing 组件。
- 基于现有 `ToolPanel` 元数据建立有序分组。
- 提供规范化查询、过滤结果和稳定 ID 查找。
- 保证本地化标签变化不影响工具身份和排序。

### `ToolSidebar`

- 构建搜索框和分组工具树。
- 处理展开、收起、过滤、键盘导航和空状态。
- 通过稳定工具 ID 发出选择事件，不直接操作 CardLayout。
- 暴露刷新本地化标签和恢复展开状态的接口。

### 共享布局常量

- 在 UI 辅助层集中维护侧栏宽度范围、行高、壳层边距和控件间距。
- 不把业务页面专用尺寸放入共享常量。

## 数据流

1. `MainFrame` 创建全部 `ToolPanel` 元数据并交给 `ToolNavigationModel`。
2. `ToolSidebar` 根据模型构建分组树，并从配置恢复展开状态和当前工具。
3. 用户选择工具后，侧栏发送稳定工具 ID。
4. `MainFrame` 根据 ID 显示对应 CardLayout 页面，更新当前位置并保存当前工具。
5. 搜索只改变侧栏可见模型；CardLayout、工具实例和当前工具状态保持不变。
6. 语言切换重新取得本地化标签并刷新可见模型；稳定 ID 和内容视图映射不变。

## 异常与边界处理

- 无效的侧栏宽度配置按 190px 至 320px 进行约束。
- 无效的当前工具 ID 回退到第一个可用工具，并写回有效配置。
- 空工具集合显示安全空状态，不访问不存在的首项。
- 搜索期间当前工具被过滤掉时，内容区继续显示当前工具；只有用户明确选择结果时才切换。
- 清空搜索后恢复搜索前的分类展开状态和当前选中项。
- 主题应用失败继续使用现有系统外观回退逻辑。
- UI 状态更新必须在 Swing Event Dispatch Thread 上执行。

## 测试策略

实现遵循测试先行。先看到目标行为测试失败，再写最小实现使其通过。

### 单元测试

- 分组保持现有顺序，工具在分组内保持现有顺序。
- 搜索可匹配稳定名称、本地化名称、分组和关键词。
- 搜索大小写不敏感，空查询返回完整导航。
- 无结果返回明确空状态，不自动选择其它工具。
- 稳定 ID 在中英文标签切换后仍定位同一工具。
- 无效工具 ID 和无效侧栏宽度得到正确回退。

### Swing 组件测试

- 侧栏选择事件只携带稳定工具 ID。
- `Ctrl+K`、方向键、`Enter` 和 `Esc` 的行为符合设计。
- 收起侧栏后展开按钮仍可聚焦和操作。
- 刷新本地化标签不会创建新的工具内容视图。
- 所有组件状态变更在 Event Dispatch Thread 上完成。

### 回归与视觉验证

- 运行完整 Maven 测试套件。
- 运行 Maven 打包，确认 Java 8 编译目标不受影响。
- 在默认窗口尺寸和 820×520 最小尺寸下检查截图。
- 分别检查一个浅色主题和一个深色主题。
- 分别检查简体中文和 English。
- 核对全部工具仍在导航中，并可切换到原有内容视图。

## 验收标准

1. 主窗口不再同时显示顶部分类页签和分类内工具列表。
2. 所有现有工具都能从统一侧栏访问，搜索能力不减少。
3. 主题、语言和状态栏能力完整保留。
4. 切换工具、搜索、主题和语言不会清空已构建工具页面的状态。
5. 侧栏可调整、可收起，状态在重启后恢复。
6. 默认尺寸和最小尺寸下不存在不可达的全局操作。
7. 键盘可以完成搜索、结果选择和导航开合。
8. 完整测试与打包通过，并完成四组主题、语言、尺寸组合的人工截图检查。

## 后续范围

完成全局工作台后，可以按独立阶段审查复杂管理页面。优先顺序建议为 Kafka、K8s 集群管理、数据库管理和 Redis 管理；这些页面应在不影响本轮工作台架构的前提下分别设计、测试和实施。
